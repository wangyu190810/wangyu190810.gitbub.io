两天时间将服务器配置完成

发表于 2014-12-24

今天最后的时候我差点就卸载了我的阿里云服务器，原因是我使用的debian的python 版本为2.7.8，但是debian的是2.6.6，我自己就安装了2.7.8安装顺利，一切我以为要结束了，但是一切的恶梦才刚刚开始

安装完成之后发现我的pip的model全部在2.6.6下面，但是我想要在2.7.8下面，但是无所谓我不去理会，因为我遇到无数这样的情况，最终决定在2.6.6
下面运行自己的软件

  • 问题开始出现了，我使用的markdowm 模块有一个函数提示说不存在,我去安装时候发现这个东西仅仅存在于python2.7版本，
  • 于是开始折腾，你们懂得，我就想要所有的pip安装的软件全部可以被python2.7.8使用，看到有使用软连接什么的？开始实践
  • 于是悲剧了，我的pip 总是告诉我说，缺少
    from pkg_resources import resource_stream
  • 我开始卸载我的pip，重新安装，但是发现不行，还这个问题，即使使用
    apt-get remove --purge
    。依旧不好使，于是我相当了，easy_install 但是他也出问题了，即使重新安装也同样不好使，最终放弃
  • 下来我想到了阿里云有ubuntu14.04的，我就想要换服务器，因为我的服务器里面几乎是空的除了一些配置文件，但是我知道自己疯了，我最终决定出去一下，不要碰电脑，要不我一定会从楼上跳下去的

最终回来时候相到了解决方案，更改系统库，这个很危险不是必要最好不要做，我该的仅仅时markdown的一个函数，

sudo python2.6 server.py

于是最终在服务器上跑了起来

####总结一下这次配置

  • 不要盲目的在一个问题上面兜圈子
  • 遇到问题停止思考，吃点东西，出去一会
  • 经常总结，不断学习

生活思考

发表于 2014-12-24

最近这周也是不知道怎么了，感觉自己做事时候总是有点急躁，比如下班回家之后我总是感到无所事事，但是有不想学习。

也许是对于未来的无法预料表示出来的担心。
可能原因如下

  • 自己的感情问题，这个问题这些年了一只都没有处理好，还是会被困扰。也许这就是命中注定
  • 工作问题，我不明白什么是工作，什么是生活，不懂得处理两个之间的交集，总以为自己会在现在的地方永远的工作下去，自己的危机意识不知道去哪里了，总以为自己是个小孩子，总是不能不处理好这些问题
  • 未来发展问题，记得自己看过无数的blog，记得他们说过的无数的话，记得他们说it行业不学习，一年你就可能出局,但是我总是打不起来精神做一些事情。

面对这些问题，我需要解决的思路和想法

  • 感情这种东西，真的很重要吗？很重要，我记得我这样说过可悲的是写的一手烂代码，还没有女朋友，更可悲的是，终于能写一手好代码了，但是还是没有女朋友，这个说明我现在不能分清楚感情和金钱那个重要（ps:我们应该努力的赚钱面包钱还是先努力的去拥有玫瑰），可能年少，可能不知未来会如何，努力生活，为了我爱着的人和那些爱我的人！
  • 工作，这个可能我要慎重的考虑一下，因为一个bat的背景还是很重要的，对于我这个很菜的人来说，但是现在又是我喜欢的工作环境，未来到底有多远谁也不知道，谁也不能预测。资本的市场，谁也不知道谁的情怀，谁也不知道，谁会被时代嘲笑，尽管只有我们自己嘲笑自己！
  • 未来，记得有个段子我以为自己很牛逼了，但是遇到一个叫做linux的坑，我就再也没有爬出来！,这个是个段子，也是人生，没有我们爬不出来的坑，只是我们想在哪个方面学习多久，如果你的公司不用linux，那样你还会继续吗？这不就逃离的了，但是我们知道我们不愿意逃离技术，不愿意离开学习。mysql我用的也是一塌糊涂，sql语句写的效率也是不忍直视，python中的装饰器，没有勇气开始学习，虽说那是必须的。

计划性的解决学习和未来：

  • 买书学习，不要再使用电子书了，没有效率，不能静下心来学习。
  • 在十月之前学习完classmethod和装饰器这两。期间使用Ctornado这个工程完成项目！
  • 注重安排自己的时间，让每天的时间更多一些。
  • 努力学习当前公司的源代码，争取快速的进入他们的开发，而不是和现在一样，虽说每天的技术都有增长，但是增长的还是有点慢！有压力才能继续学习！
